# Чек-лист безопасности Moodle

## Выполненные настройки безопасности

| N | Мера безопасности | Статус | Описание |
|---|-------------------|--------|----------|
| 1 | Смена пароля администратора | Выполнено | Установлен сложный пароль (не менее 12 символов, буквы разного регистра, цифры, спецсимволы). Пароль изменён через профиль пользователя admin. |
| 2 | Проверка прав роли "Студент" | Выполнено | Роль "Студент" не имеет административных прав. Доступны только просмотр курса и прохождение тестов. |
| 3 | Ограничение размера загружаемых файлов | Выполнено | Установлен максимальный размер загружаемого файла: 20 МБ. Настройка: Администрирование сайта -> Безопасность -> Политика сайта -> Максимальный размер загружаемого файла. |
| 4 | Защита имён пользователей | Выполнено | Включена опция "Защищать имена пользователей" (Администрирование сайта -> Безопасность -> Политика сайта). При восстановлении пароля не раскрывается информация о существовании аккаунта. |
| 5 | Требовать вход в систему | Выполнено | Включена опция "Требовать вход в систему" (Администрирование сайта -> Безопасность -> Политика сайта). Гостевой доступ отключён, все пользователи должны авторизоваться. |
| 6 | HTTPS/SSL | Не настроено | Для локального развертывания в Docker используется HTTP. В production-среде необходимо настроить SSL-сертификат. |
| 7 | Отключение неиспользуемых плагинов | Выполнено | Отключены неиспользуемые плагины: комментарии в глоссарии отключены, анонимные посты запрещены. |
| 8 | Безопасные пароли пользователей | Выполнено | Созданным пользователям (teacher_bereztova, student_bereztova) установлены временные пароли, которые необходимо изменить при первом входе. |
| 9 | Настройка политики паролей | По умолчанию | Оставлены настройки по умолчанию: минимальная длина пароля 8 символов. Рекомендуется усилить. |
| 10 | Защита от SQL-инъекций | Выполнено | Moodle использует подготовленные запросы к базе данных через API, что защищает от SQL-инъекций. Дополнительных настроек не требуется. |

## Рекомендации по усилению безопасности

1. Настроить HTTPS — использовать SSL-сертификат для шифрования трафика
2. Установить CAPTCHA — добавить проверку при регистрации новых пользователей
3. Настроить регулярное резервное копирование — автоматическое создание бэкапов базы данных и файлов
4. Обновлять Moodle — регулярно устанавливать обновления безопасности
5. Ограничить количество попыток входа — защита от подбора паролей
6. Настроить двухфакторную аутентификацию — для администраторов и учителей
